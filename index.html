about

Grounded answers, not guesses

Large language models are great at language, not at knowing the contents of your specific PDF. RAG Book Assistant closes that gap by retrieving the most relevant chunks of your document before the model ever generates an answer.

If the answer genuinely isn't in the document, the assistant says so explicitly instead of making something up.

vector_store = "ChromaDB"
embeddings = "Mistral AI"
retrieval = "MMR search"
memory_window = 6
interface = "Streamlit"
how it works

From PDF to grounded answer

Each stage prepares the document a little more before the model ever sees a question.

01
chunk

Document loading

Loads the uploaded PDF and splits it into overlapping chunks for better context continuity.

02
embed

Embedding

Converts each chunk into a vector using Mistral AI embeddings to capture its meaning.

03
store

ChromaDB storage

Persists all chunk embeddings in a local Chroma vector database for fast similarity search.

04
answer

Retrieval + generation

Pulls the most relevant chunks via MMR search and feeds them to Mistral to generate the answer.

what makes this different

It remembers the conversation

Most basic RAG demos treat every question in isolation. This one doesn't.

Rolling conversation memory

The last 6 messages are tracked in session state and passed into the prompt alongside the retrieved context, so follow-up questions stay coherent.

history += f"{'{'}role{'}'}: {'{'}content{'}'}\n"

Persistent vector database

Document embeddings are stored in ChromaDB with a persistent directory, so the knowledge base survives across sessions instead of rebuilding every time.

Chroma(persist_directory="chroma_db")
capabilities

What it does

PDF upload & parsing

Accepts any PDF, loads it directly, and prepares it for chunking without leaving the browser.

MMR-based retrieval

Uses Maximal Marginal Relevance search to fetch diverse, relevant chunks instead of redundant ones.

Context-aware chat

Combines retrieved document context with recent chat history so answers stay relevant to the ongoing thread.

Custom chat UI

Hand-styled chat bubbles in Streamlit, with user and assistant messages visually distinguished.
